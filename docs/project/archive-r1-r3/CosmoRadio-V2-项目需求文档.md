## 第二期目标

### 1. 体验效果

- 实体按键可用：具有明确交互
- 外星人查看与通话：可看到外星人当下在做什么，并且发起与外星人的对话
- 外星人任务接收：在特定情况下触发，可联动指示灯与音响发出提示

### 2. 实现方式

- Web 或者 Android APK
- 按键关联：将实体按键与网页交互操作相关联，实现实体按钮对系统的操控

### 3. 预期网页功能

**高优功能**：
- 监控浏览：可查看各个太空舱房间的情况（预制视频），通过左右摇杆进行切换操作
- 发起对话：与各个房间中的在岗的外星人发起聊天（语音输入）

**中优功能**：
- 警报提醒：紧急事件触发时，在指定房间监控室中播放警报提醒动画（预制视频）
- 进入任务：点击选择是否进入任务

### 4. 阶段交付

- 四台可供使用的预研展示设备
  - 带有完整外观（3D 打印）
  - 带有完整交互组件（与第一期相同）
  - 可实现语音输入与播放功能
- 交付时间：12 月底

## 第一期目标（整合版）

### 交付时间

- 11 月 12 日交付 2 台设备（可完整使用）
- 11 月 15 日交付 3 台设备（完整调优使用）

### 核心功能模块

**外观形态**：
- 风格：复古未来主义，Teenage Engineering 风格
- 使用方式：单手或双手握持，类似掌机使用方式；可作为挂件挂在包上，也可以放在口袋里
- 尺寸：整体直径（最宽处）10cm，屏幕直径 6cm

**硬件操作**：
- 圆形屏（彩色，无法触屏）
- 摄像头（500-800 万像素）
- 基础按键组合（录音按钮、ABCD 键）
- 麦克风及扬声器

**基础功能**：
1. 联网功能：连接 WiFi（包含热点）
2. 账号绑定：扫描二维码

**玩法雏形**：
1. 接收通知：包含文本、图片、音频、视频文件
2. 发送消息：包含语音、图片、视频文件

### 按键设计

| 按键 | 功能 |
|------|------|
| 开机键 | 开关机 |
| A 键（上） | 导航 |
| B 键（左） | 导航 |
| C 键（下） | 导航 |
| D 键（右） | 导航 |
| 录音键 | 可长按录音 |

### 用户旅程

回到家，收到来自爸爸（青少年用户）或女朋友（成人用户）的礼物，打开后显示星际信号搜索状态。

完成联网（WiFi 连接或蜂窝网络）后，收到来自外星人发布的求助任务。完成首个求助任务后，获得足够多能量，能够与外星人进行短时间通讯。在通讯过程中，外星人描述了自己面临的基本情况，并且邀请用户下载注册 App，以便更完整进行通讯。

下载 App 并注册账号后，绑定 CosmoPager 硬件终端，收到来自外星人的回信视频，感谢他参与拯救计划并说明后续会有很多需要用户帮忙的地方。

用户可以查看到目前外星人需要的地球信息类型，可以打开摄像头拍摄家庭里、室外等场景，传输给外星人。在一段时间的星际通讯后，会获得来自外星人的回复（以外星人形象录屏的形式呈现）。

随时随地带着 CosmoPager，期待收到外星人的消息或求助信号。如果现在有空，即没有在上课且有网的情况下，可以直接打开设备，帮助外星人解答问题或查看外星人给自己发布的留言内容。

这些紧急求助任务多与外星人正在进行的复刻地球实验有关，是他们在复刻过程中遇到的卡点，需要用户通过选择、语音输入的方式进行解决。

如果身边有其他持有 CosmoPager 的朋友，还可以和他们共享任务，共同完成挑战。完成任务后还可以和朋友解锁特殊征程 Special Mission。

用户可以与外星人预约通话，或者在外星人在线的情况下，发起实时通话。这些通话可能是用户的日常记录或情绪抒发，渐渐与外星人建立情感纽带。

在通话和完成一系列任务的过程中，用户可以帮助外星人在太空舱内复刻地球的自然环境和人文氛围，并且将自己参与复刻的内容放置在博物馆里，可以在 App 端进行浏览，同时也可以在 CosmoPager 上进行翻页浏览。

## 硬件预研详细需求（讨论版）

### 一、预研目标

- 为 Putopia 的 AlienAgent 项目硬件进行外观、选型和基础功能搭建
- 通过外部合作的方式快速拼接并验证可行性
- 产出一个基础可用的 Demo 版本硬件，可以实际拿在手上感受手感、整体体验链路
- 项目总周期：1 个月

### 二、预研背景

AlienAgent 是一个软硬结合的沉浸式游戏化学习/教育游戏，其中包含了 App、硬件和 IP 内容三大板块。

**硬件定位**：
- 产品名称：CosmoPager（星际寻呼机）
- 专属于 AlienAgent 世界观、游戏玩法的硬件组件
- 与 App 共享一个账号系统，共同与外星人进行通讯连接
- 具有一定玩具属性和学习属性，是整个产品链路重要的组成部分

### 三、硬件基础要求

**3.1 交互形式（已确定）**：
- 屏幕：一个较小的屏幕（用于信息展示和字幕显示）
- 物理按键：几个有限的物理按键（用于核心操作）
- 语音交互：支持语音输入和输出（麦克风 + 扬声器）
- 其他交互：建议预研团队根据世界观和成本评估，提出震动、灯效等反馈方式

**3.3 外观设计**：
- 符合"星际寻呼机"概念的基础外观形态
- 需体现科幻外星通讯设定，具有一定辨识度
- 适合手持，便于携带
- 请提供：外观设计方案（草图或 3D 建模）及设计思路说明

**3.4 连接性要求**：
- 必须支持：4G 网络或 WiFi 网络连接
- 建议评估：蓝牙连接（与手机 App 联动的必要性）

### 四、预研阶段划分

**阶段一：外观及信息接收**

硬件交付物：
- 基础外观形态的样机（可为 3D 打印外壳 + 开发板组合）
- 完成基础选型（需满足后续功能拓展，特别是语音交互）
- 实现网络连接功能（4G 或 WiFi）
- 基础音频输入输出功能

软件功能：
- 账号绑定 AlienAgent 官方账号体系
- 能够接收 AlienAgent 的推送通知信息
- 基础 UI 界面（显示通知内容）

验收标准：
- 样机可正常联网并登录账号
- 可接收并显示服务器推送的测试消息
- 外观符合世界观基调

**阶段二：互动类玩法叠加**

软件功能需求：
- 星际信号接收：定时或触发式接收来自"外星人"的信号/消息
- 语音聊天：通过语音方式与 AI 外星人进行对话交互，同时屏幕显示字幕
  - 支持语音输入（用户说话）
  - 支持语音输出（AI 回复）
  - 实时字幕显示在屏幕上
- 接收紧急任务：突发任务通知，支持震动/声音/灯效提醒，对任务进行反馈

互动玩法实现方式（二选一）：
- 方案一：通知系统 + WebView 交互（轻量化）
- 方案二：独立原生应用实现（体验更流畅）

验收标准：
- 三大核心功能均可正常运行
- 物理按键可完成基础操作流程
- 整体交互链路流畅，无明显卡顿

### 五、技术方案要求

**5.1 硬件规格选型**

> 重要约束：单台样机成本必须控制在 600 元以内

核心配置：
- 处理器/主控芯片方案
- 内存/存储容量建议
- 屏幕规格（尺寸、分辨率、类型）
- 音频模块（麦克风、扬声器/耳机接口规格）
- 电池容量及预估续航时间
- 网络模块方案（4G 模组 / WiFi 芯片）

物理参数：
- 整机尺寸范围建议（长 × 宽 × 厚）
- 重量范围
- 充电接口类型
- 是否建议增加传感器（陀螺仪、震动马达等）

成本评估：
- 单台样机成本预估（必须控制在 600 元以内）
- 小批量生产（50-100 台）的单价预估
- 量产（1000 台以上）的单价预估
- 关键零部件的供应链风险评估
- 成本优化建议

**5.2 软件开发方案**

- 操作系统选择（Android 精简版 / Linux / 其他）
- 与 AlienAgent 服务器的通讯协议建议
- 账号系统对接方式
- OTA 升级方案的可行性

**5.3 扩展性评估**

- 语音识别和语音合成的性能要求
- 拍照/扫描功能
- GPS 定位
- NFC 交互
- 其他建议的扩展功能

### 六、交付物要求

**6.1 样机数量**：
- 3 台具有完整外观且可运行阶段一 + 阶段二功能的样机
- 所有样机需功能一致，外观可接受一定差异
- 成本约束：单台样机成本 ≤ 600 元

**6.2 文档交付**：

阶段完成时交付：
- 外观设计说明：设计思路、尺寸图、材质说明
- 技术实现方案：软硬件架构图、数据流程图
- 生产可行性评估：小规模组装生产的流程和难点
- 风险评估：技术风险、供应链风险及应对方案

阶段二完成后补充（可延后提交）：
- 硬件选型方案文档：包含所选芯片/模组/屏幕等详细型号及理由
- 成本分析报告：样机成本详细清单（需 ≤ 600 元/台）、BOM 表、小批量及量产成本预估、优化建议

**6.3 可持续生产方案**：
- 提供可持续小规模组装生产的方案建议
- 明确关键零部件的采购渠道和起订量
- 评估从样机到小批量生产的改造成本和周期

### 七、项目管理

**7.1 时间规划**：
- 总周期：1 个月
- 请预研团队提供详细的项目时间表和里程碑规划

**7.2 验收标准**：

硬件验收：
- 外观符合世界观设定，具有辨识度
- 屏幕显示正常，物理按键响应准确
- 网络连接稳定，续航满足基本演示需求
- 三台样机功能一致性达到 90% 以上

软件验收：
- 账号绑定流程顺畅，无明显 Bug
- 信息推送实时性良好（延迟 < 5 秒）
- 语音输入输出功能正常，字幕显示准确
- 三大互动功能均可演示
- 基础操作流程可通过物理按键完成

体验验收：
- 手持体验舒适，重量适中
- 整体交互链路完整，符合预期
- 可作为产品概念展示使用

### 八、协作方式

- 项目启动会：明确需求细节和技术方案
- 每周进度会：同步开发进度和问题
- 随时沟通：钉钉/微信群及时响应问题
- 审核节点：方案评审 → 阶段一验收 → 阶段二验收 → 最终样机验收

### 九、知识产权

- 外观设计版权归 Putopia 所有
- 技术方案文档归 Putopia 所有
- 预研团队可保留通用技术积累

### 十、风险与应对

| 风险 | 预案 |
|------|------|
| 供应链延迟（芯片/屏幕/外壳加工） | 提前评估关键零部件交期，准备备选方案 |
| 技术实现难度超预期 | 语音功能考虑云端 + 本地双方案 |
| 网络连接稳定性问题 | 预留缓冲时间用于问题修复 |
