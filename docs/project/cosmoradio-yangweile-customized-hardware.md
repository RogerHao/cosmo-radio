### 需求总结

来源：03-31 上午会议 + 下午电话沟通 + **04-01 需求讨论会议**。

#### 1. Action Button 改进（核心）

- **现状问题**：易卡住、手感弱、行程不足
- **优先级**：稳定性（不卡） > 行程（≥3mm） > 手感（暂不追求）
- **方案方向**：键盘轴 + **6.25U 卫星轴空格键**（~10cm 长度），借鉴机械键盘蝴蝶结构
- **信号兼容**：维持 0/1 数字信号，模拟回车键，长按逻辑不变
- **进展（04-01）**：物料预计 04-02/03 到货，需 Arthur 寄 Pad 进行装配手感测试

#### 2. 结构优化

- **螺丝固定**：M2.5×10 平头内六角螺丝（比十字花更精密美观）
- **壁厚增加**：2mm + 肋板加强
- **模块化设计**：顶部模块（按钮+电子件，独立打印）、背板模块（可替换）
- **前面板斜度**：维持不变，避免打印困难
- **设备宽度（04-01）**：暂继续使用弯头 USB-C 线，两边各宽 1cm 增加安装灵活性
- **充电口**：需留出位置，可水平往里插

#### 3. NFC 支持

- 方案设计完成，详见 [nfc.md](../hardware/nfc.md)
- 模块：RC522 mini (SPI)，标签：NTAG215
- 通信协议：HID 键盘逐字符键入 `NFC:<UID_HEX>\n`
- **⚠️ 数量待定（04-01）**：杨炜乐希望 2 个 NFC 位置，Arthur 认为 1 个够用。1 个 NFC = 8 根线，2 个 GPIO 开销过大。建议一边放 NFC，另一边可做指示灯等（Arthur 认为指示灯控制麻烦，先不考虑）。待 Arthur 与杨炜乐讨论后确认。

#### 4. 外观

- Arthur 已出概念图，侧面还原效果，正面无变化
- 前面板磨砂效果保持，不贴膜
- 磁吸功能后期考虑
- 顶部板子通过螺丝拧上，两块板做成一体

#### 5. 音响与旋钮（04-01 确认）

- **音响**：使用平板自带音响，音量合适，无需外接
- **旋钮按压**：两侧 EC11 旋钮支持按压（SW 引脚会接线），但功能可选不用
- **协议**：旋钮和 NFC 协议将用独立文档定义

### 技术方案

| 模块 | 方案 | 状态 |
|------|------|------|
| 控制器 | ESP32-S3 DevKitC N16R8 + USB HID（升级，更多 GPIO） | 确定 |
| Action Button | 键盘轴 + 6.25U 卫星轴空格键，~10cm | 物料到货中，待手感测试 |
| NFC | RC522 mini (SPI)，NTAG215 标签 | 方案确定，数量待确认 (1 vs 2) |
| 外壳 | 模块化 3D 打印（主体 + 顶部 + 背板），M2.5×10 平头内六角螺丝 | 概念图已出 |
| PCB | 定制载板 (KiCad → JLCPCB)，XH2.54 即插即用 | 确定 |
| 音响 | 平板自带音响 | 确定（不外接） |
