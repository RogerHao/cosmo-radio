# OTG 一分二适配器 — 集成分析

> 标记号：YK16-09E V1（板上丝印，日期码 240919）
> 用途：让 Tab A9 同时处于 **Data Host**（接收 ESP32 的 USB HID）+ **Power Sink**（被外部充电器充电）
> 当前状态：作为内置子板已采购、已实测 OTG + 充电同时工作；**仅剩 dongle USB-A ↔ ESP32 之间的一根 USB 3.0 线**外露在主板侧

参考图：[otg-adapter-yk16-09e-photo.jpg](../assets/otg-adapter-yk16-09e-photo.jpg)

## 1. 这块小板内部到底在做什么

从照片可以看到三个端口和板间一颗小型 IC：

| 端口 | 类型 | 角色 | 走向 |
|------|------|------|------|
| 左（"Pad"） | **Type-C 公头**（焊接式 plug，已带短缆） | DFP 视角的 Power Sink，但宣告自己是 USB Host (DFP/Host) 的 Data Role | 插入 Tab A9 |
| 中（"Power"） | **Type-C 母座** | UFP/Sink → 上行接外部充电器（Source） | 充电器 |
| 右（"ESP32"） | **USB 3.0 Type-A 母座**（9 pin） | 标准 USB Host 提供给下游设备 | 经一根 USB 3.0 线连到 ESP32-S3 DevKitC 自带 USB-C |

中间那颗带丝印的 IC 是关键，它干三件事：

1. **CC 线协商**：在朝向平板的 Type-C plug 上呈现 Rp（pull-up），并通过 PD BMC（或固定 CC pattern）告诉平板"我是 Host + Sink"。这一步用普通 5.1k 上拉/下拉电阻**做不到**——`pcb/README.md` 里 6P 转接板 + SS34 的失败就是这个原因。
2. **Power Role / Data Role 编排**：让平板内部把电源开关切到 Sink 路径（充电器电直接进电池），同时数据通路保持在 Host 模式（向下游提供 USB Host）。
3. **VBUS 路由 + 反灌保护**：把充电器的 5V 经高边 MOSFET 转接给平板 VBUS，同时阻止平板侧 VBUS 倒灌进充电器或下游设备。

这颗 IC 大概率是 LDR6028/LDR6023C 一族（5V OTG + 充电控制器）或 IP2721 类，**没有从板上拆下来在显微镜下看丝印之前不能 100% 确认**。

## 2. ESP32-S3 实际只用 9 pin 中的 4 pin

| 引脚 | 用途 | ESP32 是否用 |
|------|------|-------------|
| VBUS | 5V 电源 | ✅ 走 DevKitC 的 5V 输入 |
| GND  | 地 | ✅ |
| D+ / D- | USB 2.0 高速差分对（480 Mbps） | ✅ |
| StdA_SSRX± / StdA_SSTX± / GND_DRAIN | USB 3.0 SuperSpeed 通道（5 GT/s+） | ❌ 全部空跑 |

**结论**：USB-A 一侧 9 pin 里有 5 pin 完全是浪费。把这条 USB 3.0 线消灭只需要把 4 根线（VBUS、GND、D+、D-）从 dongle 直接接到主 PCB 即可。

## 3. 三种集成路径，按风险从低到高排

### 方案 1（推荐 V4 采用）：拆 USB-A 母座，4 根短线直连主 PCB

**做法**：
1. 把 dongle 上的 USB 3.0 Type-A 母座从板上拆下来（USB-A 受电力 + 信号都在板边焊盘，9 pin 加 2-4 个机械固定脚，热风枪 30s 可下件）。
2. 露出 4 个有用焊盘：VBUS、GND、D+、D-。
3. 焊 4 根 AWG28-AWG30 信号线到主 PCB 上一个新增的 4P XH2.54 连接器（或直接 4 个焊盘 + 应力释放）。
4. 主 PCB 这一侧用一根短跳线接到 ESP32-S3 DevKitC 的 USB-C 座（保持 DevKitC 不变）；或者后续 PCB rev 把 ESP32-S3 的 GPIO19/20/5V/GND 直接走到这个 4P 连接器，整条 USB-C 线也省掉。

**优点**：
- 不动 dongle 上的 PD/CC 控制 IC，电气行为 0 变化。
- 工艺非常确定：拆一个 USB-A 座 + 焊 4 根线，半小时收工。
- USB 2.0 HS 480 Mbps 在 <10 cm 短线上完全不需要阻抗匹配/length matching。
- 任何一头坏了都好换。

**缺点**：
- dongle 仍是一块独立小板，机械固定方案要做（双面胶 / 螺丝柱 / 卡扣）。
- BOM 里仍有"成品 dongle 整片"这一项，不是真正"自研一片主 PCB 含 OTG"。

**成本**：dongle ¥15-25 + 4 根线 ¥0 + 1 个 4P 连接器 ¥0.5。比当前少一根 USB 3.0 线（~¥10）。

**适合 V4** —— 04/29 重启的 V4 范围是"技术验证 + 单套原型"，2026-05-20 前要交付。这条路径**风险几乎为零**，今天就能动手。

### 方案 2：dongle USB-A 焊盘改 2.54 排针，主 PCB 出 4P 排母

**做法**：把 USB-A 拆掉后不焊飞线，而是焊一个 4P 2.54 直插排针；主 PCB 对应位置放排母。

**优点**：可拆卸，调试期间方便单独验证 dongle 或主 PCB。

**缺点**：增加了机械应力点，长期连接可靠性略差，不如直接焊死。

**适合**：调试阶段做，到最终组装再焊死，等于"1 + 2 混合"。

### 方案 3（推迟到 V5）：把 dongle 内部电路完整复刻到主 PCB

**做法**：
1. 拆 dongle，显微镜读丝印识别 CC 控制 IC（关键步骤——丝印不清就死路一条，需要好相机 + 侧光）。
2. 找到 datasheet 后，把 IC、外围被动件、ESD/TVS 阵列、VBUS 路径 MOSFET 全部画进 KiCad 原理图。
3. 主 PCB 上加 1 个 Type-C 充电母座 + 1 根 Type-C 公头 pigtail（去往平板）。
4. 重新过 JLCPCB（5 天打板 + 3 天到货），做 PD/CC 行为回归测试。

**优点**：彻底单板化、形态最干净、可量产。

**缺点 / 风险**：
- IC 识别失败 → 整条路径作废（实物焊在板上的 SOT-23-6/QFN 丝印往往只有 2-3 字符代号，要查代号表，不一定能查到）。
- 即使识别出 IC，PD/CC 行为复刻不一定 1:1 一致；Tab A9 这类设备对 CC 时序敏感，可能要协议分析仪才能调通。
- 时间预算 ≥ 一个 PCB rev cycle，约 10-15 天，**吃光 V4 剩余周期**。

**结论**：**不要塞进 V4。** 这是 V5（如果客户后续要小批量量产）才动的活。文档里把这条路径标为"未来工作"即可。

## 4. 给 V4 的具体决定

| 项 | 决定 |
|----|------|
| OTG dongle | **保留为内置子板**（YK16-09E V1） |
| dongle ↔ ESP32 之间的 USB 3.0 线 | **消灭**：拆 USB-A 母座 → 4 根飞线 → 主 PCB 4P XH2.54 连接器 |
| 主 PCB 上是否新增 PD/CC 控制 IC | **不新增** |
| 主 PCB 上是否新增 Type-C 母座（充电输入） | **不新增**（充电仍走 dongle 上自带的 Type-C 母座） |
| 朝向平板的 Type-C 公头线 | **保留 dongle 自带的**（它就是焊在 dongle 板上的 plug + 短缆） |
| 后续工作（V5+） | 显微镜读 IC 丝印 → 复刻入主 PCB |

## 5. 在动手之前要做的小件事（10 分钟）

1. **拍一张 dongle 中央 IC 的高分辨率照片**（手机 macro 或 USB 显微镜，侧光读丝印）。即使本期不复刻，IC 型号也是 V5 文档的金字塔基。把照片存成 `docs/assets/otg-adapter-yk16-09e-ic-marking.jpg`。
2. **量一下 dongle PCB 物理尺寸**（含 Type-C plug 短缆出板方向 + 长度 + 三个端口的开口位置），决定外壳里给它留哪一块平面 + 螺丝柱。
3. **确认 USB-A 母座下件后的 4 个焊盘极性**（VBUS、D-、D+、GND，从左到右是 USB-A 标准定义，但实物板可能镜像）。可以用万用表通断从 USB-A 金属壳和 4 个 pin 之间反查。

## 6. 主 PCB 改动清单

如果 V4 PCB 还没下单，加这几项；如果已经下单，飞线接也行：

- 新增 1 个 **J5（dongle USB 数据接入）** = 4P XH2.54 立式直插，位于 PCB 边缘靠近 ESP32-S3 DevKitC USB-C 那一侧。
- 信号顺序建议 `VBUS / D- / D+ / GND`（与 USB-A 物理顺序一致，方便排查）。
- VBUS 走线宽度 ≥ 0.6 mm（承载 ≤2 A 充电电流过路）。
- D+ / D- 走差分对，宽度 ~0.2 mm，间距 ~0.13 mm，长度匹配差 < 5 mm（USB 2.0 HS 容差很宽）。
- 在 J5 上的 D+ / D- 各预留一个不焊接的 ESD 二极管位（USBLC6-2 类或 0402 TVS），现版可以 0Ω 跳过。

## 8. 一句话总结

**能集成，但分两步走。** V4 阶段把 dongle 当成"焊死的内置子板"，仅消灭 USB 3.0 长线（4 根飞线即可，零电气改动）；V5 阶段再考虑识别 dongle 上的 CC 控制 IC、整板复刻。这样 V4 deadline (2026-05-20) 守得住，长期可控性也保住了。
